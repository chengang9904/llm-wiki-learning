# 核实 Wiki 来源引用

## 步骤

1. **提取引用**：read_wiki_page 后收集两处引用：frontmatter 的 `sources:` 列表、
   正文「关键实现」里的 `（来源：`路径:行号`）`。
2. **逐条比对**：对每条引用 read_file 打开对应行段（前后多读 5 行上下文），
   判断页面陈述与代码现状是否一致。
3. **区分失效类型**（不同类型不同处置）：
   - **行号漂移**：事实还对，代码挪了位置 → 记新行号；
   - **事实过时**：代码改了语义（函数改名/逻辑变了）→ 记差异点；
   - **来源消失**：文件/函数已删除 → 记为死引用；
   - **一致**：明确记「已核实」，这也是有价值的结论。
4. **只记录不修改**（除非当前任务就是修复）：核实结果写进回答的「不确定事项」，
   或（s12 起）用 flag_issue 落到 issues.json。

## 纪律

- 不允许「页面写得挺合理所以大概没问题」——合理性不是证据，比对才是；
- 一次核实控制在 5-10 条引用内，多了分给子 Agent；
- 引用行号超出文件长度直接判失效，不用读内容（lint 的 cite_out_of_range 同款规则）。
