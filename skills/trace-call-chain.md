# 追踪跨模块调用链

## 原则

调用链追踪最容易失败的方式是「一头扎进去顺着读」——上下文很快被中间文件塞满。
正确姿势：**先分段，每段独立查清，最后拼接**。

## 步骤

1. **切段**：按架构层或进程边界把链路切成 2-5 段
   （如：HTTP 入口 → service 编排 → 外部服务调用 → 持久化）。
   用 todo_write 把段落列成计划。
2. **锚点定位**：每段先 grep 一个可靠锚点（handler 方法名、client 构造函数、
   asynq 任务名），拿到 `文件:行号` 再展开。
3. **分段派发**：每段交给一个 spawn_subagent，任务描述必须自包含：
   写明段的起点锚点、期望终点、要返回的证据格式。
4. **汇合**：把各段的 call_chains 首尾相接；接不上的缝隙（如事件/队列异步边界）
   明确标注「异步断点」——asynq 入队和消费是两条链，不要假装连续。
5. **Wiki 优先**：动手 grep 前先 search_wiki——workflows/ 类页面可能已有现成链路。

## 结论格式

```
A.func（文件:行号）→ B.func（文件:行号）→ [异步: 队列名] → C.func（文件:行号）
```
每个箭头都要能答「在哪一行发起的调用」。
